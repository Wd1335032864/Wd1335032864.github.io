图的基本算法

发表于 2022-05-03 更新于 2022-05-04 分类于 数据结构与算法
本文字数： 7.7k 阅读时长 ≈ 7 分钟
遍历，最小生成树，最短路径

图的定义

图是由顶点的有穷非空集合V ( G ) 和顶点之间边的集合E ( G )组成，表示为G=（V，E）。V中元素的个数为G中顶点的个数，也是图的阶数，E中元素的个数为G的边数。

图不能是空图，顶点集不能为空 ，至少有一个顶点，边集可以为空。

基本概念

阶：图G中点集V的大小称作图G的阶
有向图：
无向图：
重复边：无向图中为两顶点相同的边，有向图中为起点终点相同的边
简单图：①不存在重复边;②不存在顶点到自身的边（自环）
多重图：不满足简单图的其中一点
完全图：也称简单完全图 无向完全图和有向完全图都为完全图
稠密图：有很少边或孤的图 通常用邻接矩阵实现
稀疏图：有较多边或孤的图 通常用邻接表实现
子图：设有两个图G=（V，E）和G’=（V’，E’），若V’是V的子集，且E’是E的子集，则称G’是G的子图
生成子图：设有两个图G=（V，E）和G’=（V’，E’），若V’等于V，且E’是E的子集，则称G’是G的生成子图
连通：无向图中，从一个顶点到另一顶点有路径存在 则称这两个顶点连通
连通图： 图G中任意连各个顶点都是连通的，则称图G为连通图
连通分量：无向图中极大连通子图称为连通分量
强连通图：在有向图中，若从顶点v到顶点w和从顶点w到项点v之间都有路径,则称这两个顶点是强连通的，任意两点皆如此，则称该有向图为强连通图
强连通分量：有向图中的极大强连通子图称为强连通分量
度：无向图中顶点依赖的边的条数
入度：有向图中以顶点v为终点的边的条数
出度：有向图中以顶点v为起点的边的条数

图的遍历

BFS

BFS是分层查找的过程，类似于树的层序遍历，即每向前遍历一步会访问一批节点，不存在回溯的过程，所以会要借助队列实现

  1. 访问起始点v
  2. 之后，依次访问v的各个未曾访问过的邻接点
  3. 再分别从这些邻接点出发，访问他们的未被访问过的邻接点；
  4. 直至图中所有顶点都被访问过为止

DFS

DFS类似于树的先序遍历，每次遍历访问一个节点，直到不能再继续访问再回溯。

  1. 访问图中的某一起始顶点v

  2. 由v出发，访问与v相连，并且没有被访问过的顶点w

  3. 访问与w相邻但是并没有被访问过的顶点x

  4. 当不能向下访问的时候，依次退回到最近被访问的顶点

  5. 访问直至所有顶点均被访问过

拓扑排序

定义

设G = ( V , E ) G=(V,E)G=(V,E)是一个具有n个顶点的有向图，V中的顶点序列V1 , V2 , . . . Vn ，若Vi到Vj有一条路径，则在顶点序列中，Vi要在Vj之前，我们称这样的顶点序列为一个拓扑序列
拓扑排序，就是对一个有向图构造拓扑序列的过程
在一个表示工程的有向图中，用顶点表示活动，用边表示活动之间的优先关系，这样的有向图为顶点表示活动的网，我们称为AOV网
通常会用拓扑排序来处理AOV网问题 AOV网的拓扑排序可能会有多个

算法

  1. 从AOV网中选择一个没有前驱的顶点并输出。

  2. 从网中删除该顶点和所有以它为起点的有向边。

  3. 重复1和2直到当前的AOV网为空或当前网中不存在无前驱的顶点为止。如果输出顶点数少了，就说明这个网存在环(回路)，不是AOV网。

最小生成树

生成树

如果连通图G的一个子图是一棵包含G的所有顶点的树，则该子图称为G的生成树
生成树是连通图的包含图中的所有顶点的极小连通子图

最小生成树

当图上的边带有权值时，我们将构造这个最小总代价的生成树称为最小生成树

Prim算法

​ a. 利用邻接矩阵存储dist[i][j]两点 i 和 j 之间的距离；
    b. 用cost[i]来表示 最小生成树集合 和 非最小生成树 中的点 i 的最小距离，当cost[i] = 0代表 i 就是 最小生成树 集合中的顶点。
    c. 由于是生成树，所以顶点 0 一定在树上，初始化cost[i]就是 0 和 i 的距离（因为 最小生成树集合 目前只有0）。
    d. 从cost[i]中寻找一个值不为零（因为值为零表示是最小生成树集合中的点）且最小的顶点u，那么 cost[u]一定是 最小生成树上的边。于是，u也成了 最小生成树上的点。
    e. 然后，继续用u去更新cost[i]，即更新 最小生成树集合 和 非最小生成树的点 之间的距离，回到 d 继续迭代计算。

( 1 ) cost[i]表示 当前最小生成树集合 和 当前非最小生成树 中的点 i 的最小距离，当cost[i] = 0代表 i 就是 当前最小生成树 集合中的顶点；
( 2 ) ret用来存储最小生成树边权之和，初始化为 0；
( 3 ) 从cost[i]中寻找一个值不为零（因为值为零表示是最小生成树集合中的点）且最小的顶点u，那么 cost[u]一定是 最小生成树上的边。于是，u也成了 最小生成树上的点；
( 4 ) 整个查找过程完成，直接返回最小生成树的边权总和；
( 5 ) 将当前边cost[u]加入最小生成树；
( 6 ) 将当前点u加入最小生成树；
( 7 ) 继续用u去更新cost[i]，即更新 最小生成树集合 和 非最小生成树的点 之间的距离；

Kruscal算法

​ 采用 并查集 实现，算法描述如下：
    a. 将图中所有的边按照三元组 (u,v,w) 来存储。
    b. 然后按照第三关键字 w 将所有边进行递增排序；
    c. 顺序取边，并且判断当前边 (u,v) 的两个顶点 u 和 v 是否在同一个集合。如果不在，则这条边就是 最小生成树 上的边，权值累加，合并两个点；如果在，则这条边舍去；
    d. 反复迭代取边，直到总共取了 n−1 条边，则算法结束。

（1）并查集的初始化；
( 2 ) 带路径压缩的并查集查找操作；
( 3 ) 并查集的合并操作；
( 4 ) 定义边三元组；
( 5 ) 按照边权从小到大排序的回调函数；
( 6 ) 对所有边进行排序；
( 7 ) 初始化所有结点的并查集信息；
( 8 ) 将当前边加入到最小生成树中；
( 9 ) 如果边数等于n−1 则找到解，直接返回；
